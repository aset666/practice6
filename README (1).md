# ML FastAPI Docker — Iris Species Predictor

A minimal end-to-end example of training a machine learning model and serving it as a REST API with **FastAPI**, containerised with **Docker**.

---

## Project Structure

```
ml-fastapi-docker/
├── main.py           # FastAPI application
├── train.py          # Model training script
├── model.joblib      # Saved model (generated by train.py)
├── requirements.txt  # Python dependencies
├── Dockerfile        # Container instructions
└── README.md
```

---

## Setup — Virtual Environment

```bash
# Create virtual environment
python -m venv venv

# Activate (Linux / macOS)
source venv/bin/activate

# Activate (Windows)
venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt
```

> To deactivate: `deactivate`

## Quick Start (local, without Docker)

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Train the model

```bash
python train.py
```

This creates `model.joblib` in the project root.

### 3. Run the API

```bash
uvicorn main:app --reload
```

The server starts at **http://localhost:8000**.

### 4. Test the endpoints

**Root endpoint:**
```bash
curl http://localhost:8000/
# {"message":"ML API is running"}
```

**Prediction endpoint:**
```bash
curl -X POST http://localhost:8000/predict \
     -H "Content-Type: application/json" \
     -d '{"features": [5.1, 3.5, 1.4, 0.2]}'
```

Expected response:
```json
{
  "predicted_class_id": 0,
  "predicted_class_name": "setosa",
  "probabilities": {
    "setosa": 1.0,
    "versicolor": 0.0,
    "virginica": 0.0
  }
}
```

**Interactive Swagger docs:**  
Open **http://localhost:8000/docs** in your browser.

---

## Docker Deployment

### 5. Build the Docker image

```bash
docker build -t ml-fastapi-docker .
```

### 6. Run the container

```bash
docker run -p 8000:8000 ml-fastapi-docker
```

Then test the same endpoints above — they work identically inside the container.

## API Reference

| Method | Path       | Description                      |
|--------|------------|----------------------------------|
| GET    | `/`        | Health check                     |
| POST   | `/predict` | Predict Iris species from features |

### POST `/predict` — Request body

| Field    | Type          | Description                                              |
|----------|---------------|----------------------------------------------------------|
| features | `float[4]`    | [sepal_length, sepal_width, petal_length, petal_width] cm |

### POST `/predict` — Response

| Field                | Type     | Description                              |
|----------------------|----------|------------------------------------------|
| predicted_class_id   | `int`    | 0 = setosa, 1 = versicolor, 2 = virginica |
| predicted_class_name | `string` | Human-readable class label               |
| probabilities        | `object` | Per-class probability scores             |

---

## Model Details

- **Dataset**: Iris (150 samples, 4 features, 3 classes)
- **Algorithm**: Random Forest (100 estimators)
- **Test accuracy**: ~96–100 %
